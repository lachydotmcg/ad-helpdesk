FAQ
Common questions
Does it work with on-premises Active Directory? +

Yes. AID Helpdesk is built specifically for on-premises Windows Active Directory. A lightweight agent runs on your AD server and connects outbound to the cloud — no inbound firewall rules or VPN changes needed.

Is Azure AD / Entra ID supported? +

Not yet — Azure AD / Entra ID support is on the roadmap. The current version targets on-premises AD environments running Windows Server. Join the waitlist on signup to be notified.

How secure is it? +

All communication is over HTTPS. The agent polls the cloud using a per-tenant API key — your AD is never directly exposed to the internet. The AI assistant also actively flags impersonation attempts and suspicious requests before any action is taken. Every write operation is logged in a tamper-evident audit trail.

What AD actions can the AI assistant perform? +

Unlock accounts, reset passwords, enable/disable accounts, add/remove group membership, move users between OUs, create new user accounts, and retrieve user info. More actions are added regularly.

Can my whole team use it? +

Yes. The Pro plan includes up to 5 dashboard users per tenant with role-based access — admin (full control) or viewer (read-only). Additional seats coming soon.

How do I get started? +

Create a free account, copy your API key from Settings, drop it into agent-config.json on your Windows server, and run python agent.py. The whole setup takes under 5 minutes.
